Заявки на приобретение МТР — приложение, предназначенное для создания, учета, планирования и отслеживания заказов на материально-технические ресурсы в электронном виде.

Поля:

Все поля выделенные желтым цветом обязательны для заполнения.

 

  • Номер — формируется автоматически. Порядок нумерации сквозное по организации, т.е. с начала года формируется номер равное 1, в начале следующего года нумерация начинается заново.
  • Организация — организация, для которой производится закупка оборудования, по умолчанию выводится организация из настроек пользователя.
  • Подразделение — наименование подразделения, для которого осуществляется закупка.
  • Исполнитель от подразделения — ответственный сотрудник, инициатор заявки, ставит финальную подпись(выполнено-работа завершена)
  • Основание — документ, на основе которого происходит закупка МТР, документ должен предварительно создан и доступен в архиве документов, необязательное поле. Документом может являться план.
  • Описание документа в архиве — необязательное поле, краткое описание документа, которое будет отображаться в программе «Архив документов».
Работа с позициями:

Добавление: Добавляет новую позицию в таблицу МТР

Редактирование: Редактирует уже созданную позицию в текущей строке

Копировать: Добавляет новую позицию на основе уже имеющихся данных

Удалить: Удаляет позицию в текущей строке

Порядок прохождения заявки:

Прохождение заявки происходит в несколько этапов:

  1. Сотрудник составляет заявку и пересылает его всем ответственным лицам для согласования. Если к заявке предшествовал план или служебная записка, то этот документ указывается в поле "Основание"
  2. Все ответственные сотрудники согласовывают(подписывают) документ
  3. После подписания документа, формируется счет на оплату либо договор поставки, либо приложение к договору и привязывается к документу "Заявки МТР" как вытекающий документ.
  4. На основе документов п.3, бухгалтерией формируются платежное поручение или swift(для валюты)
  5. После появления МТР на складе, к документу указанному в п.3 привязывается документ "Товарно-транспортная накладная" как вытекающий документ.

Правила подписания документа:

Подписание согласовывается по стандартной схеме:

Шаг 1. Сотрудник составляет заявку и пересылает его всем ответственным лицам для подписания.

Шаг 2. Все сотрудники, утверждающие данный документ ставят свою подпись.

Шаг 3. После завершения всех закупок, ответственный сотрудник сотрудник подписывает документ как выполнено-работа завершена.

Документ имеющий хотя бы одну подпись является не редактируемым.
Если необходимо сделать правки в подписанный документ необходимо:
  1. Удалить все подписи
  2. Отредактировать документ
  3. Разослать всем ответственным сотрудникам для ознакомления и повторного подписания.
Распределение документов по позиции:

Термины:

Документы-основание оплаты — могут быть, счет, договор, либо приложение к договору. Данный вид документа отображается в столбце "счета, договоры".
Документы оплаты — могут быть платежым поручением либо swift. Данный вид документа отображается в столбце "Платежи"
Документ ТТН — могут быть только товарно-транспортная накладная. Данный вид документа отображается в столбце "ТТН"
Не распределенные документы — документы-основание оплаты, которые еще не распределены по позициям.
Распределенные документы — документы-основание оплаты, которые еще распределены по позициям.
Drag-and-drop (транск. "Драг Энд Дроп"; в переводе с английского означает буквально тащи-и-бросай) — способ оперирования элементами интерфейса.
Способ реализуется путём «захвата» (нажатием и удержанием левой кнопки мыши) отображаемого на экране компьютера объекта,
и перемещения его в другое место («бросания» - отпусканием левой кнопки мыши).
В документе "Заявка МТР" операцию Drag-and-drop можно произвести между документами-основаниями оплаты и конкретной позицией МТР


После создания документа "Заявка МТР" и наличия хотябы одной подписи, документ переходит в статус нередактирумый.
В этом состоянии документу возможно добавлять связанные с ним документы.
Документы доступные для связывания находятся в поле в таблице Не распределенные документы либо Распределенные документы.
Данные таблицы доступны в двух местах: в блоке "Изменяемая часть документа" и в диалоговом меню(вызывается путем нажатия на колонку "счета, договора" со значком )
Список таблиц формируется из связи: "Заявка МТР как документ основание" -> "Документы-основание оплаты как вытекающее"
В диалоговом меню(вызывается путем нажатия на колонку "счета, договора" со значком ) для упрощения связи документа МТР и Документом существует поле "Документ:"
при выборе документа из этого поля и подтверждения связи, в таблице появится новый нераспределенный документ.

Для пользователя необходимо распрелить только Документы-основание оплаты все остальные документы распределяются по следющей схеме:
  • Платежи - должна быть связь "Документы-основание оплаты как основание" -> "Документы оплаты как вытекающее"
  • Документ ТТН - должна быть связь "Документы-основание оплаты как основание" -> "ТТН как вытекающий"
Выдача МТР:

После появления хотя бы одного документа ТТН появляется записи об доступных ресурсах к выдаче.
Для выдачи необходимо нажать на кнопку «Выдать».
Далее появится документ «Акт приема-передачи» и всплывающее окошко с надписью «Выдать ресурсы».
Необходимо выбрать склад, с которого выдается ресурс, получатель и сам ресурс(количество). После нажать кнопку «Выдать».
